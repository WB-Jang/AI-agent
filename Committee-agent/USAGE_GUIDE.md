# 사용 가이드 (Usage Guide)

## 📄 문서 오타 검수 기능

### 1단계: 파일 업로드 및 API 키 입력

1. **OpenAI API Key 입력**: 왼쪽 상단의 비밀번호 입력란에 OpenAI API 키를 입력합니다.
2. **파일 업로드**: "검수할 파일 업로드 (Word/PDF)" 영역에서 `.docx` 또는 `.pdf` 파일을 드래그하거나 "Browse files" 버튼을 클릭하여 업로드합니다.

### 2단계: 문서 미리보기

파일이 업로드되면 왼쪽 패널에 문서 내용이 미리보기로 표시됩니다.
- Word 문서(.docx)의 경우: 전체 텍스트가 표시됩니다.
- PDF 문서(.pdf)의 경우: 추출된 텍스트가 표시됩니다.

### 3단계: AI 검수 시작

1. 오른쪽 패널의 **"오타 검수 시작"** 버튼을 클릭합니다.
2. AI가 문서를 분석하는 동안 진행률 표시줄이 나타납니다.
3. 분석이 완료되면:
   - **왼쪽 패널**: 오타가 감지된 문장이 빨간색으로 하이라이팅됩니다.
   - **오른쪽 패널**: 상세한 수정 제안이 섹션별로 표시됩니다.

### 4단계: 검수 결과 확인

#### 왼쪽 패널 (문서 미리보기)
```
⬇️ 오타가 감지된 문장이 빨갛게 표시됩니다.

[섹션 제목]
일반 텍스트... [오류 부분]빨간색으로 표시된 오타[/오류 부분] ...더 많은 텍스트
```

#### 오른쪽 패널 (AI 수정 제안 요약)

각 섹션별로 펼치기/접기가 가능한 카드 형태로 표시됩니다:

```
📋 AI 수정 제안 요약

📌 서론
┌─────────────────────────────────────┐
│ [삭제 하고] ➡️ [삭제하고]            │
│                                     │
│ 💡 수정 이유: 동사와 보조동사는    │
│ 띄어쓰지 않습니다.                  │
└─────────────────────────────────────┘

📌 1. 기본 원칙
┌─────────────────────────────────────┐
│ [피하고 ,쉬운] ➡️ [피하고, 쉬운]    │
│                                     │
│ 💡 수정 이유: 쉼표 앞에는 공백이   │
│ 없어야 하며, 쉼표 뒤에 공백을      │
│ 넣어야 합니다.                      │
└─────────────────────────────────────┘
```

### 결과 해석

- ✅ **녹색 체크마크**: 해당 섹션에 오타나 수정 사항이 없습니다.
- ❌ **빨간색 취소선**: 오류가 있는 원본 텍스트
- ✔️ **녹색 배경**: 수정 제안 텍스트
- 💡 **파란색 박스**: 수정 이유 설명

## 🔄 PDF 일괄 변환 기능

### 1단계: 변환할 폴더 지정

1. **"PDF 일괄 변환"** 탭을 클릭합니다.
2. 변환할 파일이 있는 폴더 경로를 입력합니다 (기본값: 현재 작업 디렉토리).

### 2단계: 변환 시작

1. **"일괄 변환 시작"** 버튼을 클릭합니다.
2. 시스템이 지정된 폴더에서 `.docx`, `.doc`, `.pptx`, `.ppt` 파일을 자동으로 찾습니다.

### 3단계: 변환 진행 확인

실시간으로 변환 상태가 표시됩니다:
- ⏳ **진행 중**: 현재 변환 중인 파일명
- ✅ **성공**: 성공적으로 변환된 파일
- ❌ **실패**: 변환 실패한 파일 및 에러 메시지

### 4단계: 결과 확인

- 변환된 PDF 파일은 `pdf_output` 폴더에 저장됩니다.
- 원본 파일명과 동일한 이름으로 `.pdf` 확장자가 붙습니다.

예시:
```
원본 폴더/
├── document1.docx
├── presentation1.pptx
└── pdf_output/
    ├── document1.pdf
    └── presentation1.pdf
```

## 💡 팁과 주의사항

### 오타 검수 기능
- **API 비용**: OpenAI API 사용량에 따라 비용이 발생합니다.
- **문서 크기**: 매우 큰 문서는 처리 시간이 오래 걸릴 수 있습니다.
- **지원 언어**: 현재 한국어에 최적화되어 있습니다.
- **섹션 구분**: 볼드체 텍스트를 섹션 제목으로 인식합니다.

### PDF 변환 기능
- **시스템 요구사항**: Linux 환경에서 LibreOffice가 설치되어 있어야 합니다.
- **한글 폰트**: 한글이 포함된 문서의 경우 한글 폰트가 설치되어 있어야 정상적으로 표시됩니다.
- **변환 시간**: 파일 크기와 수에 따라 변환 시간이 달라집니다.
- **임시 파일**: 변환 중 생성되는 임시 파일(`~$`로 시작)은 자동으로 제외됩니다.

## 🔧 문제 해결

### 오타 검수가 작동하지 않는 경우
1. OpenAI API 키가 올바르게 입력되었는지 확인
2. API 키에 충분한 크레딧이 있는지 확인
3. 인터넷 연결 상태 확인

### PDF 변환이 실패하는 경우
1. LibreOffice 설치 확인: `libreoffice --version`
2. 파일 권한 확인
3. 디스크 공간 확인

### 문서가 제대로 파싱되지 않는 경우
1. Word 문서 형식이 `.docx`인지 확인 (구형 `.doc`는 제한적 지원)
2. 문서에 손상이 없는지 확인
3. 다른 문서로 테스트해보기
